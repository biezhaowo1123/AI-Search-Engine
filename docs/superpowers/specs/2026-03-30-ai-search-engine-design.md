# AI 搜索引擎 - 设计文档

**日期：** 2026-03-30  
**版本：** 简化版 - 仅文字搜索

---

## 1. 概述

**核心目标：** 构建一款简单的 AI 文字搜索引擎，用户输入查询，返回搜索结果 + AI 生成的答案摘要。

**约束条件：** 无需 VPN/代理，在国内完美运行

**用户规模：** 中小型产品，支持数千到数万用户

---

## 4. 核心功能模块

### 4.1 搜索服务 (Search Service)

- 接收用户查询请求
- 调用国内搜索 API（神马/百度）获取搜索结果
- 结果清洗、去重、格式化
- 调用 AI 摘要服务生成答案

### 4.2 缓存服务 (Cache Service)

- 搜索结果缓存（TTL: 30分钟）
- AI 摘要缓存（TTL: 1小时）
- 热门查询预热

### 4.3 认证与限流服务 (Auth & Rate Limit Service)

- API Key 管理
- 请求限流

## 5. API 设计

### 5.1 搜索接口

```
POST /api/search
Headers:
  Authorization: Bearer <api_key>

{
  "query": "string",
  "page": 1,
  "page_size": 20
}

Response:
{
  "results": [
    {
      "title": "string",
      "url": "string",
      "snippet": "string",
      "date": "string"
    }
  ],
  "ai_summary": "string",
  "cached": boolean
}
```

### 5.2 认证接口

```
POST /api/auth/apikey
{
  "email": "string"
}

Response:
{
  "api_key": "string",
  "rate_limit": {
    "per_minute": 60,
    "per_hour": 1000
  }
}
```

---

## 6. 数据流程

1. 用户输入查询 → API 网关（鉴权 + 限流）
2. 检查 Redis 缓存
3. 未命中 → 调用神马搜索 API
4. AI 摘要服务生成答案（调用 Minimax）
5. 结果存入缓存 + 返回用户
