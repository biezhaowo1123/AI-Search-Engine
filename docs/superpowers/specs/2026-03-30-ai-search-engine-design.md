# AI 多模态搜索引擎 - 设计文档

**日期：** 2026-03-30  
**方案：** 方案 B - 全功能原型（国内优化版）

---

## 1. 概述

**核心目标：** 构建一款支持文本、图片、视频、音频全模态搜索的 AI 搜索引擎，通过国内可用的第三方搜索 API 获取数据，结合多模态 AI 模型处理和向量数据库检索，提供更智能的搜索体验。

**约束条件：** 无需 VPN/代理，在国内完美运行

**用户规模：** 可扩展商业产品，支持百万级用户

## 4. 核心功能模块

### 4.1 搜索服务 (Search Service)

- 接收用户查询请求
- 调用国内搜索 API（百度/神马）获取原始结果
- 结果清洗、去重、格式化
- 调用 AI 摘要服务生成答案摘要

### 4.2 多模态处理服务 (Multimodal Service)

| 功能 | 模型 | 输入 | 输出 | 供应商 |
|------|------|------|------|--------|
| 图片理解 | Minimax GLM-Vision / ERNIE-ViLG 2.0 / 通义 VL | 图片 URL | 图片描述 + 向量 | Minimax/百度/阿里 |
| 音频转写 | Minimax ASR / 讯飞星火 / 阿里 ASR | 音频文件/URL | 文字 + 向量 | Minimax/讯飞/阿里 |
| 视频理解 | 多模态融合 | 视频 URL | 关键帧 + 字幕向量 | 自建 + 百度 |
| 文本嵌入 | 智谱 ChatGLM-Embedding | 文本 | 768 维向量 | 智谱 |

### 4.3 向量检索服务 (Vector Search Service)

- 存储：接收多模态向量，存入腾讯云向量数据库
- 检索：余弦相似度匹配，返回 top-k 结果
- 支持多向量融合（图片+文本+音频加权）

### 4.4 缓存服务 (Cache Service)

- 查询结果缓存（TTL: 1小时）
- 用户会话存储
- 热门查询预热

## 5. API 设计

### 5.1 搜索接口

```
POST /api/search
{
  "query": "string",
  "filters": {
    "modality": ["text", "image", "video", "audio"],
    "date_range": { "start": "date", "end": "date" }
  },
  "page": 1,
  "page_size": 20
}
```

### 5.2 以图搜图接口

```
POST /api/search/image
{
  "image_url": "string",
  "query": "string (optional)"
}
```

### 5.3 答案摘要接口

```
POST /api/ask
{
  "query": "string",
  "context": ["url1", "url2"]
}
```

## 6. 数据流程

### 文本搜索流程
1. 用户输入查询 → API 网关
2. 检查阿里云 Redis 缓存
3. 未命中 → 调用百度搜索 API / 神马搜索
4. AI 摘要服务生成答案（调用 Minimax / 文心一言 / 通义千问）
5. 结果存入缓存 + 返回用户

### 多模态搜索流程
1. 用户上传图片/音频/视频
2. 多模态服务提取特征向量（Minimax GLM-Vision / 文心 VL / 讯飞 ASR）
3. 腾讯云向量数据库相似度检索
4. 融合排序 + AI 摘要
5. 返回结果
