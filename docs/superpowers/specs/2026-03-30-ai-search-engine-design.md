# AI 多模态搜索引擎 - 设计文档

**日期：** 2026-03-30  
**方案：** 方案 B - 全功能原型（国内优化版）

---

## 1. 概述

**核心目标：** 构建一款支持文本、图片、视频、音频全模态搜索的 AI 搜索引擎，通过国内可用的第三方搜索 API 获取数据，结合多模态 AI 模型处理和向量数据库检索，提供更智能的搜索体验。

**约束条件：** 无需 VPN/代理，在国内完美运行

**用户规模：** 可扩展商业产品，支持百万级用户

## 4. 核心功能模块

### 4.1 搜索服务 (Search Service)

- 接收用户查询请求
- 调用国内搜索 API（百度/神马）获取原始结果
- 结果清洗、去重、格式化
- 调用 AI 摘要服务生成答案摘要

### 4.2 多模态处理服务 (Multimodal Service)

| 功能 | 模型 | 输入 | 输出 | 供应商 |
|------|------|------|------|--------|
| 图片理解 | Minimax GLM-Vision / 通义 VL | 图片 URL | 图片描述 + 向量 | API 调用 |
| 音频转写 | Minimax ASR / 阿里 ASR | 音频文件/URL | 文字 + 向量 | API 调用 |
| 视频理解 | 百度视频理解 API | 视频 URL | 关键帧 + 字幕向量 | API 调用 |
| 文本嵌入 | 智谱 Embedding / Minimax Embedding | 文本 | 768 维向量 | API 调用 |

### 4.3 向量检索服务 (Vector Search Service)

- 存储：接收多模态向量，存入腾讯云向量数据库
- 检索：余弦相似度匹配，返回 top-k 结果
- 支持多向量融合（图片+文本+音频加权）

### 4.4 缓存服务 (Cache Service)

- 查询结果缓存（TTL: 1小时）
- 用户会话存储
- 热门查询预热
- 缓存失效：主动失效 + TTL 过期

### 4.5 认证与限流服务 (Auth & Rate Limit Service)

- API Key 管理（用户维度）
- 请求限流（每用户/每分钟/每小时）
- 读写权限分离

### 4.6 可观测性服务 (Observability Service)

- 日志：阿里云日志服务 SLS
- 指标：阿里云 Prometheus
- 告警：钉钉/企微通知
- 链路追踪：阿里云 ARMS

## 5. API 设计

### 5.1 搜索接口

```
POST /api/search
Headers:
  Authorization: Bearer <api_key>

{
  "query": "string",
  "filters": {
    "modality": ["text", "image", "video", "audio"],
    "date_range": { "start": "date", "end": "date" }
  },
  "page": 1,
  "page_size": 20
}
```

### 5.2 以图搜图接口

```
POST /api/search/image
Headers:
  Authorization: Bearer <api_key>

{
  "image_url": "string",
  "query": "string (optional)"
}
```

```

Headers:
  Authorization: Bearer <api_key>
```

## 6. 数据流程

### 文本搜索流程
1. 用户输入查询 → API 网关（鉴权 + 限流）
2. 检查阿里云 Redis 缓存
3. 未命中 → 调用神马搜索 API
4. AI 摘要服务生成答案（调用 Minimax）
5. 结果存入缓存（TTL: 1小时）+ 返回用户

### 多模态搜索流程
1. 用户上传图片/音频/视频 → 存储到阿里云 OSS
2. 多模态服务提取特征向量（API 调用）
3. 腾讯云向量数据库相似度检索
4. 融合排序 + AI 摘要
5. 返回结果

### 数据索引流程（后台）
1. 爬虫抓取网页内容
2. 内容审核服务过滤（百度内容审核 API）
3. 文本分块 + 嵌入向量生成
4. 向量存入腾讯云向量数据库
5. 元数据存入 MySQL（阿里云 RDS）

## 10. 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 搜索 API 成本超预期 | 高 | 高 | 1) 多家搜索 API 比价（神马/百度/360） 2) 自建垂类爬虫（技术文档/新闻） 3) 用户配额制度 |
| AI 能力调用限制 | 中 | 中 | 多家供应商备份（Minimax/通义/智谱），自动切换 |
| 向量数据库扩展性 | 低 | 中 | 前期按需扩展，腾讯云支持在线扩容 |
| 第三方 API 稳定性 | 中 | 高 | 降级策略：搜索 API 故障时返回缓存或报错 |
| 内容合规风险 | 中 | 高 | 接入百度内容审核 API，自动过滤违规内容 |
